# Use source-relative seconds as canonical time

All editorial timestamps use Source Time: decimal seconds from the first decodable displayed frame of the selected video stream, with the exclusive end at that stream's last displayed frame end. Persistence stores non-negative integer microseconds and half-open intervals, while APIs and prompts may render decimal seconds; signed audio/container offsets remain conversion metadata. We reject floating-point identity and independent coordinate systems because rounding or drift would make correct proposals and labels produce incorrect playback or exports, accepting explicit stream transforms in exchange.
