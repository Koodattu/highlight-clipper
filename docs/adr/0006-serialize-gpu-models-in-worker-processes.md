# Serialize GPU models in worker processes

The application never owns a CUDA model in its main process; an atomic Windows named mutex grants one cross-instance GPU Lease at a time to a disposable child process for transcription, transcript embeddings, or semantic evaluation. Qwen embeddings require CUDA and take the same mutex; llama.cpp requires full target and drafter layer offload and disables automatic fitting. A stage releases the lease only after the owned process tree exits. We reject in-process load/unload, CPU model fallback, and memory-delta-only ownership because allocator state and unrelated applications are not reliable signals, accepting model startup overhead and explicit insufficient-VRAM failures for deterministic ownership and cancellation. ASR and embeddings record before/after WDDM use; llama.cpp persists worker PID, startup/evaluation duration, effective context, and VRAM before/loaded/delta. Named-mutex ownership, peak VRAM, post-unload llama VRAM, and an enforced external-headroom policy remain diagnostic hardening rather than implemented blockers.
