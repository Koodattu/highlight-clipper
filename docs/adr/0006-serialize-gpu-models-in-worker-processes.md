# Serialize GPU models in worker processes

The application will never own a CUDA model in its main process; an atomic Windows named mutex grants one cross-instance GPU Lease at a time to a disposable child process for transcription, embedding, or semantic evaluation. A stage releases that lease only after the owned process tree exits; return toward a recorded WDDM memory baseline is a warning/headroom diagnostic because unrelated desktop processes can change total usage. We reject in-process load/unload and memory-delta-only blocking because allocator state and unrelated applications are not reliable ownership signals, accepting model startup overhead in exchange for deterministic ownership, cancellation, and recovery.
