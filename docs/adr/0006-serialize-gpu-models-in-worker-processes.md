# Serialize GPU models in worker processes

The application never owns a CUDA model in its main process; an atomic Windows named mutex grants one cross-instance GPU Lease at a time to a disposable child process for transcription, a future GPU embedding profile, or semantic evaluation. The current Qwen embedding worker is CPU-only and therefore does not take the mutex. A stage releases the lease only after the owned process tree exits. We reject in-process load/unload and memory-delta-only ownership because allocator state and unrelated applications are not reliable signals, accepting model startup overhead for deterministic ownership and cancellation. ASR records before/after WDDM use; llama.cpp persists worker PID, startup/evaluation duration, effective context, and VRAM before/loaded/delta. Named-mutex ownership, effective GPU-layer offload, peak VRAM, post-unload llama VRAM, and an enforced external-headroom policy remain diagnostic hardening rather than implemented blockers.
